# AGENTS.md

## What This Is

This is a personal Neovim configuration using vim.pack for plugin management (not lazy.nvim). Work within this structure to add features, fix issues, or refactor safely.

## Plugin System & Structure

- **Plugin manager**: Built-in vim.pack (Neovim 0.12+), not lazy.nvim
- **Plugin spec format**: Defined in `types.lua` — each plugin is a `PluginSpec` with `src`, `data` (containing `config` fn and `keys` array), `name`, and optional `version`
- **Plugin discovery**: `lua/plugins/helpers.lua:M.get_plugins()` returns all plugins; categories auto-load via `require()` calls
- **Categories**: `treesitter`, `lsp`, `code-completions`, `ai`, `git`, `ui`, `ux`, `misc`, `theme`
- **Plugin setup**: `lua/plugins/init.lua:M.setup_plugin()` runs config functions and registers keymaps after pack installation
- **Each category is a directory** with an `init.lua` that returns a `PluginSpec[]` table, e.g. `lua/plugins/ai/init.lua`

## Key Files & Entrypoints

- `init.lua` → `lua/config/init.lua` — loads options, keymaps, plugins in that order
- `lua/config/options.lua` — Neovim settings
- `lua/config/keymaps.lua` — global keymaps (also see per-plugin keymaps in their `data.keys`)
- `lua/config/utils/` — helper modules (keymap, theme utilities)
- `lua/values/` — centralized constants and configuration values (ui, languages, workflow)
- `lua/plugins/theme.lua` — theme plugin spec + theme switching logic
- `nvim-pack-lock.json` — plugin versions (auto-generated by vim.pack)

## Adding a Plugin

1. Create or update a plugin file in the appropriate category directory (e.g., `lua/plugins/lsp/newplugin.lua`)
2. Return a `PluginSpec` object with `src` (git URL, may use `gh:` shorthand) and `data` (optional config fn, optional keys array)
3. Add it to the category's `init.lua` in the return table
4. Plugin auto-loads on next Neovim start

Example:
```lua
-- lua/plugins/lsp/newplugin.lua
--- @type PluginSpec
return {
  src = "gh:user/repo",
  data = {
    config = function()
      require("plugin").setup({...})
    end,
    keys = {
      { mode = "n", key = "<leader>x", command = ":PluginCmd<CR>", desc = "..." },
    },
  },
}
```

## Dependencies & Requirements

- **Neovim**: 0.12+
- **External tools**: tree-sitter, ripgrep, fd, node/npm, python3/pip, lua/lua-rocks, cargo (for Rust plugins)
- **No lockfile in git**: vim.pack uses `nvim-pack-lock.json` locally; plugin versions may vary

## Theme System

- Available themes: catppuccin, everforest, kanagawa, onedark, tokyonight, nordic, sonokai, gruvbox, base16
- Active theme set in `lua/values/ui.lua:M.theme`
- Theme switching via `lua/plugins/theme.lua` logic

## Plugin-Specific Notes

- **codecompanion, copilot, copilot-chat**: AI plugins configured in `lua/plugins/ai/` with keymaps using shared prefix system
- **Mason/lspconfig**: Language servers installed and configured via Mason; individual server configs in `lua/plugins/lsp/lspconfig.lua`
- **Blink.cmp**: Completion engine; related config in `lua/plugins/code-completions/`
- **Treesitter**: Syntax highlighting and parsing; parsers managed separately in `lua/plugins/treesitter/`

## Keymap Conventions

- Keymaps use a prefix system: check `lua/values/constants/prefix.lua` and `lua/values/constants/shared.lua` for leader and prefix mappings
- Helper: `config.utils.keymap.map()` and `with_prefix()` used for consistency
- Per-plugin keymaps go in the plugin's `data.keys` array; global keymaps in `lua/config/keymaps.lua`

## Configuration Values

- UI constants: `lua/values/ui.lua` (logo, title, theme)
- Language-specific settings: `lua/values/languages.lua`
- Workflow constants: `lua/values/workflow.lua`
- Shared constants (prefixes, descriptions, commands): `lua/values/constants/shared.lua`

## Debugging / Commands

- `:PackListAll` — lists all installed plugins (defined in `lua/plugins/init.lua`)
- `vim.print()` used for debugging output
